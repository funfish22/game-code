# 液態玻璃設計系統

## 設計變數基準

依背景與內容密度調整下列範圍：

| 變數 | 常用範圍 | 用途 |
| --- | --- | --- |
| 玻璃覆色 | 8%–22% 白色或深色 | 透明材質 |
| 降級覆色 | 78%–94% 不透明 | 無模糊時維持可讀性 |
| 邊框亮度 | 18%–40% 淺色 | 定義玻璃邊緣 |
| 模糊 | 12–28px | 擴散後方背景 |
| 飽和度 | 115%–170% | 保留透過玻璃的色彩 |
| 圓角 | 14–32px | 建立表面層級 |
| 位移強度 | 3–8 | 小型元件的輕微折射 |
| 陰影模糊 | 12–36px、低透明度 | 分離表面而不顯厚重 |

優先使用一個強調色搭配中性玻璃；第二個強調色只用於狀態或危險操作。

## 高度層級

### 第一層：內嵌

用於清單列、次要控制與輸入框內槽。

- 透明度差異最小
- 圓角較小
- 陰影極少
- 位於另一個玻璃表面內時，通常不再套背景模糊

### 第二層：浮動

用於主容器、卡片、導覽與工具列。

- 中等模糊與邊框對比
- 柔和外陰影
- 一道節制的內側高光

### 第三層：覆蓋

用於對話框、彈出選單與重要暫時表面。

- 覆色最不透明
- 邊緣定義最清楚
- 必須與下方內容保持足夠對比

## 元件指引

### 按鈕

- 主要操作比周圍玻璃更不透明。
- 使用色彩、字重與位置表達重要性，不只依靠光暈。
- 滑入或按下位移不超過 1–2px。

### 工具列

- 將工具列浮在內容上方，形成獨立功能層。
- 手機上的主要分頁導覽固定在底部安全區上方；寬螢幕再視情境改用頂部或側邊。
- 相似操作或影響同一內容區域的控制放在同一群組。
- 每個功能群組共用一個玻璃背景；無關群組以空隙分開。
- 同一群組內保持圖示與標籤形式一致。
- 常見操作使用熟悉符號，並提供可存取名稱。
- 內容可從工具列下方滑過，但需以覆色、模糊或邊緣漸層維持可讀性。
- 外層群組與內層控制使用同心圓角。

### 流動選取器

- 使用一個獨立選取塊在選項間移動。
- 以元素幾何位置更新 `translate`、寬度與高度。
- 移動中只短暫橫向拉伸約 1.05–1.12 倍。
- 記錄前一個選項時，可依移動方向調整拉伸的變形原點。
- 文字與圖示保持在選取塊上方，不跟著折射。

### 輸入框

- 使用安靜的內嵌表面與清楚邊框。
- 保留原生文字選取、自動填入與輸入法功能。
- `:focus-visible` 使用不依賴透明度的實線外框。

### 清單

- 不要讓每一列都成為厚重玻璃卡片。
- 使用一個主容器，加上節制的列分隔或輕量內嵌覆色。

### 彈出視窗

- 頁面遮罩需足以分離層級。
- 彈出視窗覆色比一般卡片更不透明。
- 不要只靠背景模糊表達前後層。

## 背景場景

玻璃後方必須有可辨識的材質。可使用：

- 兩至三個徑向漸層
- 搭配覆色的本機圖片
- 模糊的偽元素或絕對定位色彩光暈

裝飾層需設定 `pointer-events: none` 並排除於無障礙樹之外。

## 響應式指引

- 小螢幕降低模糊與背景裝飾複雜度。
- 空間不足時允許主容器延伸至視窗邊緣。
- 觸控目標至少約 44px。
- 固定底部工具列需納入 `env(safe-area-inset-bottom)`。
- 清單底部留白需讓最後一項完整捲到工具列上方。
- 避免固定高度遮住放大文字或瀏覽器工具列造成的可視區變化。
